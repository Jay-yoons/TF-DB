1. 컨트롤 파일(Control File) 관련
기존 컨트롤 파일 백업 (원본 안전 확보)
컨트롤 파일 복제 및 대상 서버로 복사
복사된 컨트롤 파일 경로 확인 및 권한 설정(읽기/쓰기 가능)
SQL> SHOW PARAMETER control_files;

# OS에서 컨트롤 파일 복사
$ cp /u01/app/oracle/oradata/ORCL/control01.ctl /backup/control01.ctl
$ scp /backup/control01.ctl standby-server:/u01/app/oracle/oradata/ORCL/



2. 리두 로그 파일(Redo Log) 관련
리두 로그 그룹 및 멤버 상태 확인 (현재 구조 파악)
리두 로그 파일 복제 및 대상 서버로 복사
복사본 리두 로그 크기 및 위치 적절히 설정
select group#, thread#, sequence#, bytes/1024/1024 mbyte, members, status 
from v$log 
order by thread#, group#;


현재 로그 그룹 내의 파일정보를 확인한다.
select * from v$logfile;

리두 로그 그룹을 하나 더 추가해준다.
alter database add logfile group 3
('/u01/app/oracle/oradata/testdb/disk1/redo03a.log',
'/u01/app/oracle/oradata/testdb/disk2/redo03a.log') size 30m;

복구 테스트 시나리오 준비 및 점검 (실제 복구 절차 검증)

백업 방법 (RMAN 스크립트, 명령어 등 구체적 절차)

복구 시나리오 (실제 상황별 복구 절차)

검증 및 테스트 (백업 검증, 복구 테스트 결과)
